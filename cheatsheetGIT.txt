Przykładowy commit:
touch hello.py

git status 	# Daje aktualny stan towjego repozytorium lokalnego i ci powie, czy jakiś plik w danym katalogu jest śledzony
git add hello.py 	# Sprawia, że git widzi jakiś plik
git commit -m "Komentarz" 	# Dodaje plik do gita, ale tego lokalnego
git push origin [nazwa galezi, np. main]	 # Updateuje plik do GitHuba, 'origin' to jest adres
git pull                  	 # Pobiera najnowsze zmiany z GitHuba
git log                    	# Pokazuje historię commitów
git branch                 	# Pokazuje aktualną gałąź
git checkout -b nowa_galaz 	# Tworzy i przełącza na nową gałąź

git stash -> schowa zmiany tymczasowo
git stash pop -> wywali te zmiany, które zostały zapisane tymczsowo

git --version # sprawdza wersję gita

Repozytaria zdalne:
git push origin <nazwa.brancha>
git pull
git fetch
git restore . # cofasz zmiany z tego co edytowałeś ale nie dodałeś do statging area
git stash # gdy chcesz opóscić branch bez commitów
git stash pop # cofasz to co masz w stash'u
git revert # tworzy nowy commit ze starego commitu 
git rm f four.txt # usuwa jakiś plik pomimo, że nie został dodany ( normalnie się nie ma możliwości)
git rm -cached four.txt # usuwa plik z trakingu w repo git'a
git rm -r <Folder> # usuwa jakieś pliki 

git log # oglądanie histori commitów 
git log --oneline # oglądanie histori commitów w wersji skróconej

Branchinig:
git branch # pokazuje branch w jakim się znajduję
git branch <nazwa>  # tworzy branch o nazwie 
git checkout <nazwa> # przenosi do innego brancha
git merge main -m "Załączam jakiś tam feature" # załącza zmiany z maina do brancha w którym się znajduje 
git merge <nazwa> -m "Załączam zmiany z brancha <nazwa> do brancha w którym się znajduję 
git checkout <commit id > # przechodzi do poprzedniego commita
git Rebase # zmienia bazwowy branch do którego wporowadzanie są zmiany ( trzeba mieć własyny branch)
git remote add origin <URL> # tworzy zadalne repo z lokalnego
